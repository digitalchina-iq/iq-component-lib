概述

简易的开关滑块，开关默认为关闭。可以双向绑定，默认为true，当不是true或false时可以自定义checked-value

何时使用

  1. 需要表示开关状态/两种状态之间的切换时；

  2. 和 checkbox的区别是，切换 switch 会直接触发状态改变， 而 checkbox 一般用于状态标记，需要和提交操作配合。

checked-value

[checked-value]为可自定义的值。跟ngModel双向绑定的值相等，则开关开启。

<iq-switcher [(ngModel)]="fromSwitcher" [checked-value]="'aaa'" ></iq-switcher>

unchecked-value

[unchecked-value]为可自定义的值。

<iq-switcher [(ngModel)]="fromSwitcher" [checked-value]="'aaa'" [unchecked-value]="'0'" ></iq-switcher>

(ngModelChange)

根据ngModelChange来实现改变后的事件

<iq-switcher (ngModelChange)="modelChange($event)" [(ngModel)]="fromSwitcher" [checked-value]="'aaa'" [unchecked-value]="'0'" ></iq-switcher>
参数 说明 类型 可选值 默认值 是否必选
checked-value 绑定的值等于这个值，开关为开启状态 any true 否
unchecked-value 绑定的值等于这个值，开关为关闭状态 any false 否

(ngModelChange)

模板语法，可监听双向绑定的值变化
