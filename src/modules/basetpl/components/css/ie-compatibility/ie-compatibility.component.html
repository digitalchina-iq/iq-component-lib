概述

将ie10出现的兼容性问题进行总结，使页面能够完美的在ie浏览器显示!

弹出框table在鼠标滑过(hover)时出现黑线

原因:   table宽度不够或者未设置宽度导致其出现错位

解决:   为table增加宽度

盒模型真实宽度过长或过短

过长原因:   父元素中的子元素宽度之和大于父元素的宽度.

解决:   使子元素宽度之和等于父元素的宽度.

过短原因:   ie10无法识别自定义指令标签是否为块元素标签,默认为行元素标签.

解决:   将自定义指令标签设置为: display:inline-block;

Input输入框添加placeholder内容后颜色显示问题

解决: input:-ms-input-placeholder color:#888; //IE

鼠标滑过(Hover)时边框消失

原因:  boder-collapse:collapse;此时如果可能,边框会合并为一个单一的边框。会忽略 border-spacing 和 empty-cells 属性.

解决:  border-collapse: separate;此时边框会被分开。不会忽略 border-spacing 和 empty-cells 属性.

浮动ie产生的双倍距离

原因: FF 和 IE 对 BOX 理解的差异导致设为浮动float的div在ie下margin加倍.

解决:   对浮动元素添加 display:inline.

#box{{{float:left; width:100px; margin:0 0 0 100px; //这种情况之下IE会产生200px的距离 display:inline; //使浮动忽略}}}

IE8以及IE8以上的版本的Hack写法

问题:  ie浏览器显示有时会与谷歌等浏览器显示不一致(如滚动条宽度),此时需要针对ie浏览器设置特殊的css样式.

解决:  在css属性值后添加\0,写出只能ie8以上版本才能识别的css代码.

Selector[property: value\0;]

慎用\0的CSS Hack

在Web页面制作中尽量不要使用CSS Hack来处理兼容问题。因为任何浏览器下出现渲染不一致都极有可能是我们自己的结构或样式不符合W3C的某些要求，或者说违背了浏览器的某些规则而造成的，所以我们应该尽量通过结构或CSS的修改来达到各浏览器渲染一致效果，除非到了万不得已情况下，才考虑CSS的Hack。
